# Классификатор лесных грибов

Нейросеть на базе EfficientNetB1 (transfer learning + fine-tuning), которая
классифицирует фотографии грибов по 9 распространённым родам. Обучена на
датасете [Mushrooms Classification — Common Genus's Images](https://www.kaggle.com/datasets/maysee/mushrooms-classification-common-genuss-images)
(~6700 изображений, 9 классов).

Проект переработан из оригинального Jupyter-ноутбука (`notebooks/Mushrooms_original.ipynb`,
писался и запускался в Google Colab) в набор обычных Python-скриптов, чтобы:
- модель можно было обучить и запускать на своём компьютере, не только в Colab;
- любой пользователь мог загрузить свою фотографию и получить предсказание
  через простое веб-окно (без командной строки и Python).

> **Дисклеймер.** Это учебный проект. Точность модели не гарантирует
> безопасность — **никогда не используйте её как единственный способ
> определить, съедобен ли гриб**. При сомнениях обращайтесь к специалисту-микологу.

## 1. Установка

Понадобится **Python 3.10–3.11** (TensorFlow 2.15–2.16 не поддерживает 3.12+
на некоторых платформах — если у вас 3.12, проверьте актуальную версию
TensorFlow под вашу ОС).

```bash
# 1. Скачать репозиторий
git clone https://github.com/<ваш-логин>/mushroom-classifier.git
cd mushroom-classifier

# 2. Создать и активировать виртуальное окружение
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt
```

Если у вас есть GPU (NVIDIA) — TensorFlow подхватит его автоматически при
наличии установленных CUDA/cuDNN. Без GPU всё тоже будет работать, но
обучение (не распознавание!) займёт заметно больше времени.

## 2. Обучение модели с нуля

Если вы хотите обучить модель сами (а не просто пользоваться готовыми
весами — см. раздел 3):

### 2.1 Настройка Kaggle API

Датасет скачивается автоматически через `kagglehub`, но для этого нужен
API-ключ Kaggle:

1. Зарегистрируйтесь на [kaggle.com](https://www.kaggle.com), если ещё не зарегистрированы.
2. Перейдите в **Account → Settings → API → Create New Token** — скачается
   файл `kaggle.json`.
3. Положите его в `~/.kaggle/kaggle.json` (Linux/macOS) или
   `C:\Users\<имя>\.kaggle\kaggle.json` (Windows).

### 2.2 Подготовка данных и запуск обучения

```bash
python src/prepare_data.py   # скачает датасет и разобьёт на train/val (80/20)
python src/train.py          # обучение: ~10 эпох "головы" + до 20 эпох fine-tuning
```

По умолчанию обучение останавливается раньше (`EarlyStopping`), если
качество на валидации перестаёт расти. После обучения появятся:

- `models/mushroom_efficientnet_b1.h5` — финальные веса модели;
- `models/best_efficientnetb1.h5` — веса лучшей эпохи (по `val_loss`);
- `models/class_indices.json` — соответствие "индекс → название класса"
  (обязателен для распознавания, без него предсказания будут бессмысленны).

**Время обучения:** на GPU уровня Colab T4 — около 1–2 часов. На CPU может
занять от нескольких часов до суток — если GPU нет, рекомендуем обучать в
Google Colab (бесплатный GPU) или на Kaggle Notebooks, используя тот же
`notebooks/Mushrooms_original.ipynb`, и затем скопировать файлы из раздела
выше себе в `models/`.

## 3. Распознавание своих фотографий

Обученная модель обязана лежать в `models/mushroom_efficientnet_b1.h5`
вместе с `models/class_indices.json` (см. раздел 2, либо если вам прислали
готовые веса — просто положите оба файла в папку `models/`).

### Вариант А — веб-интерфейс (рекомендуется для обычных пользователей)

```bash
python app.py
```

В терминале появится адрес вида `http://127.0.0.1:7860` — откройте его в
браузере. Дальше просто перетащите своё фото гриба в окно — модель покажет
топ-5 наиболее вероятных видов с процентом уверенности.

### Вариант B — командная строка

```bash
python src/predict.py путь/к/вашему/фото.jpg
python src/predict.py путь/к/вашему/фото.jpg --top 5
```

## 4. Список классов

Модель различает 9 родов грибов из исходного датасета:

`Agaricus`, `Amanita`, `Boletus`, `Cortinarius`, `Entoloma`, `Hygrocybe`,
`Lactarius`, `Russula`, `Suillus`

(точные названия и их порядок сохраняются в `models/class_indices.json`
после обучения).

## 5. Технические детали

- **Архитектура:** EfficientNetB1 (предобучена на ImageNet) + Dropout →
  Dense(512, swish) → BatchNorm → Dropout → Dense(softmax).
- **Вход:** изображения 240×240, RGB.
- **Балансировка классов:** oversampling редких классов до размера самого
  большого + `class_weight="balanced"` при обучении.
- **Аугментация:** повороты, сдвиги, зум, горизонтальное отражение.
- **Обучение в 2 этапа:** сначала обучается только "голова" (базовая сеть
  заморожена, lr=1e-3), затем размораживаются верхние слои базовой сети и
  дообучаются с lr=1e-4 (fine-tuning).

## 6. Частые проблемы

- **`ModuleNotFoundError: No module named 'tensorflow'`** — не выполнен
  `pip install -r requirements.txt` или активировано не то виртуальное окружение.
- **Ошибка авторизации Kaggle (`401`/`403`) при `prepare_data.py`** — проверьте,
  что `kaggle.json` лежит по правильному пути и имеет права `600`
  (`chmod 600 ~/.kaggle/kaggle.json`).
- **Модель на CPU считает медленно** — это нормально для обучения; для
  однократного распознавания фото (predict.py / app.py) даже CPU работает
  за секунды.
- **`FileNotFoundError` на `models/mushroom_efficientnet_b1.h5`** — модель
  ещё не обучена (или веса не скопированы в `models/`), см. раздел 2 или 3.
